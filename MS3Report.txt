# MS3 Progress Report: ocaml-autodiff

**Team:** Richie Xue (rx77), Colby Bittner (cb954), Jayden Lim (jcl399), Eric Chen (ec_936)
**Date:** May 9, 2026

## Vision

Our vision is to build a robust, idiomatic OCaml library for automatic differentiation (autodiff) that supports both reverse-mode and forward-mode techniques. Since MS1, we have sharpened our focus toward providing a developer-friendly CLI for rapid prototyping of differentiable expressions and a baseline training engine for simple machine learning models (e.g., linear regression). We aim to provide clear visualizations of computation graphs and verified gradient accuracy through numerical checking, making the library useful for both educational purposes and small-scale optimization tasks.

## Summary of Progress

Between MS2 and MS3, our team successfully transitioned from a working prototype to a polished and tested system ready for final submission. We expanded our test suites significantly across modules, including expression evaluation, forward-mode AD, gradient checking, the trainer, and Graphviz export — and integrated Bisect_ppx for code coverage reporting. The trainer module was substantially extended with loss history tracking and support for loading real CSV datasets, and we added two sample datasets to validate linear regression convergence. The Graphviz export module was refactored with improved op-focused node labeling and regression-tested against the new styling. The command-line interface was overhauled to support the full range of library features, including forward-mode differentiation and gradient checking commands, with improved error messaging throughout. Finally, all public-facing modules received complete OCamldoc documentation, and the README was updated to reflect current build instructions, CLI usage, and visualization workflows.
